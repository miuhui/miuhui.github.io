HTTP与HTTPS

发表于 2019-02-17 阅读次数：
本文字数： 414 阅读时长 ≈ 1 分钟

HTTP

HTTP超文本传输协议，是客户端与浏览器数据请求与响应的协议，HTTP采用明文进行信息传输，不提供任何方式的信息加密，很容易被攻击者窃取导致信息泄露。同时，客户端与服务器都无法验证对方身份。

HTTPS

HTTPS安全套接字层超文本传输协议，简单而言就是HTTP协议的安全版。它在HTTP的基础上增加了SSL层，作用是：1、对浏览器与服务器之间传输的信息进行加密；2、验证服务器的身份。

不同点

  • 端口不同。HTTP使用80端口，HTTPS使用443端口。
  • 安全性。HTTP明文传输信息，HTTPS对信息进行加密。

HTTPS缺点

  • SSL证书要钱
  • 采用HTTPS，连接服务器端资源占用增大。
  • HTTPS握手阶段比较耗时，有损用户体验。

HTTPS工作流程

  1. TCP三次握手
  2. 客户端验证服务器
  3. DH算法协商对称加密算法与密钥，hash加密密钥
  4. SSL安全加密隧道
  5. 网站以协商后的加密算法与密钥对数据进行加密，以协商后的Hash加密密钥对数据完整性进行验证。
