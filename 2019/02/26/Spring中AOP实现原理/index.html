Spring AOP实现原理

发表于 2019-02-26 阅读次数：
本文字数： 504 阅读时长 ≈ 1 分钟

Spring AOP是通过AOP框架创建AOP代理实现的，AOP代理分为静态代理与动态代理，静态代理以AspectJ为代表，动态代理以Spring AOP为代表。

Spring AOP无需对JAVA源码进行编译，采用运行时在内存中动态创建“代理类”来生成AOP代理。AOP代理类中包含了目标对象的所有方法，但是AOP方法与目标对象方法存在差异，AOP方法在特定位置进行了增强处理，并回调了目标对象的方法。

Spring AOP对AOP代理类的处理原则

  • 如果目标对象的实现类实现了接口，Spring AOP将使用JDK动态代理来生成AOP代理类。
  • 如果目标对象的实现类没有实现接口，Spring AOP将采用CGLib来生成AOP代理类。

JDK动态代理

JDK动态代理只能针对实现了接口的类生成代理，利用拦截器(拦截器需要实现invocationHandler)与反射生成一个实现代理接口的匿名类。

CGLib动态代理

CGLib动态代理是基于继承的，通过加载代理对象类的Class文件，修改其字节码生成子类来处理，覆盖其中的方法，实现增强，因为是采用的继承，所以类和方法最好不要声明为final，对于final的类与方法是无法继承的。
