JAVA异常体系

发表于 2019-02-26 阅读次数：
本文字数： 558 阅读时长 ≈ 1 分钟

JAVA异常都是对象，是Throwable类的子类。Throwable有两个子类，Error与Exception。

ERROR

Error错误，是应用程序运行时很严重的问题，一般为JVM出现的问题，例如outOfMemory。出现Error时JVM一般都会中止运行。

EXCEPTION

程序本身可以处理的异常。

运行时异常
RuntimeException类及其子类。例如NullPointerException、IndexOutOfBoundsException等，这些异常都是不检查异常，即Java编译器不会检查这些异常，也就是说，当程序中出现这类异常，且没有使用try-catch进行捕获，也没有使用throws字句抛出，也会编译通过。

非运行时异常
RuntimeException以外的异常，原则上程序必须处理这些异常，如果不处理，程序不能通过编译。

处理

除了Error和RuntimeException的其他异常，java要求程序必须进行处理，要么使用try-catch进行捕获，要么使用throws子句抛出。

try-catch-finally语句

一般情况下，finally语句都会被执行，除了以下几种情况

  • finally语句块中发生了异常
  • 在前面的代码中使用System.exit()退出了程序
  • 程序所在线程死亡
  • CPU被关闭
