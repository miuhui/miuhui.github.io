String VS StringBuilder VS StringBuffer

发表于 2019-01-12 阅读次数：
本文字数： 522 阅读时长 ≈ 1 分钟

String

String一旦被赋值后就不能不修改，每次改变String的值，都会新建一个对象，存储在字符串常量池中。

StringBuffer

StringBuffer是可变的，意味着可以改变对象中的值。通过StringBuffer创建的对象存储在堆中。

StringBuffer中包含与StringBuilder相同的方法，区别是StringBuffer的方法都加上了synchronized，不容许两个线程同时访问相同的方法，StringBuffer是线程安全的。StringBuffer对象可以通过toString()方法转换为String对象。

StringBuilder

StringBuilder与StringBuffer类似，对象存储在堆中，对象的值可以被改变。StringBuffer与StringBuilder主要的区别是StringBuilder不是线程安全的。

非多线程环境下，应该使用StringBuilder。
